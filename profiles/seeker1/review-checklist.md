# AI Review Checklist — seeker1

> Auto-generated by Product Builders. Use as custom instructions for AI review tools
> or as a human review checklist for PRs from non-engineer contributors.

## Tech Stack Compliance

- [ ] Only uses approved languages: TypeScript, SQL, CSS
- [ ] Only uses approved frameworks: next, react
- [ ] No new dependencies introduced without approval

## Security

- [ ] No hardcoded secrets, API keys, or passwords
- [ ] No `.env` files or credentials committed
- [ ] All user inputs validated and sanitized
- [ ] Auth patterns follow project conventions (supabase)
- [ ] SQL queries are parameterized (no string concatenation)
- [ ] Error messages do not leak internal details



## Code Quality

- [ ] Passes linter (eslint)
- [ ] Follows existing naming conventions
- [ ] No dead code, unused imports, or commented-out code

## Testing

- [ ] Tests included for new functionality
- [ ] Existing tests pass
- [ ] Test framework: vitest

## Error Handling

- [ ] Errors handled using exceptions pattern
- [ ] No silent error swallowing

## Git & PR

- [ ] PR description explains what and why
- [ ] Changes are scoped to a single concern

## UI / Design System

- [ ] Uses shadcn components — no custom replacements
- [ ] Styling follows tailwind approach
